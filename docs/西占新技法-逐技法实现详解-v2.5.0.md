## 1. 行星年龄 Planetary Ages（纯前端）

**①概述**：托勒密人生七阶——固定年龄带各由一古典行星主管，叠加本命星落座。
**②算法**：`PLANETARY_AGES`：月0–4｜水4–14｜金14–22｜日22–41｜火41–56｜木56–68｜土68+。当前年龄=(now−birth)/365.2422，落带标 active。
**③后端**：无。`utils/planetaryAges.js`：`PLANETARY_AGES` + `buildPlanetaryAges(chartObj,asOf)` + `buildPlanetaryAgesSnapshotText`。
**④前端**：`components/astro/AstroPlanetaryAges.js`（AstroChart + 七带列表 + 自管快照）。Tab `行星年龄/planetaryages`。
**⑤AI**：key=`planetaryages`，段头 `[行星年龄（Ages of Man）]`，入 PLANET_INFO Set。挂载随盘同步式（恒可用，故进测试 `expectedByTechnique`）。
**⑥验证**：preview 七带 0-4…68+ 显示、tab 粘住；140 测试绿。

## 3. 行星弧 Planetary / Lunar Arcs（双圈盘）

**①概述**：同太阳弧，推进弧改用所选天体（默认月亮）的二次推运移动量。
**②算法**：`solararc.compute` 弧源 `const.SUN`→`arcSource`，`rotation=daychart.get(arcSource).lon-chart.get(arcSource).lon`。
**③后端**：`solararc.py compute(..., arcSource=const.SUN)`（**默认 SUN 字节级等价**，回归已验）。`perpredict.py` 加 `getPlanetaryArc/getPlanetaryArcByDate`（默认 MOON）。`webpredictsrv.py` 加 `planetaryarc`。Java `PredictiveController /planetaryarc` + `getParams` 加 `arcSource` + `AstroHelper.getPlanetaryArc` + 常量 `PlanetaryArc`。**重编 jar(javap 验)**。
**④前端**：`components/astro/AstroPlanetaryArc.js`（克隆 AstroSolarArc 双圈盘 + 天体下拉 + async snapshot）。Tab `行星弧/planetaryarc`。
**⑤AI**：key=`planetaryarc`，段头 `[行星弧（Planetary Arc）]`，async builder fetch `/predict/planetaryarc`(默认月亮弧)。
**⑥验证**：preview 双圈盘(svg 92)+弧源选择器；太阳弧(arcSource=Sun)回归一致(svg 110)。

## 4. 129年系统 129 Year System（年限序列·仿 firdaria）

**①概述**：真·129**年**体系（非十年大运的 129 月递归）：七政各管其小年，一轮 129 年，含子限。
**②算法**：`MINOR_YEARS={日19,月25,土30,木12,火15,金8,水20}`(Σ=129)；起始星=sect 光体(日昼/月夜)；SEQUENCE 序铺开，每主期等分 7 段子限(仿 firdaria `avg=years/7`)。**succession 序实验性、待校。**
**③后端**：**新模块** `astropy/astrostudy/yearsystem129.py`（镜像 firdaria.py）；`perpredict.getYearSystem129`；`helper.getPredictivesObj` 随盘 bundle 加 `'yearsystem129'`（**无 endpoint/无 Java**）。
**④前端**：`components/astro/AstroYearSystem129.js`（克隆 AstroFirdaria 表，读 `chartObj.predictives.yearsystem129` + 自管快照）。Tab `129年系统/yearsystem129`。
**⑤AI**：key=`yearsystem129`，段头 `[129年系统表格]`，挂载随盘同步式(读 predictives)。
**⑥验证**：preview 主限/子限表渲染；与十年大运并存不混淆(年 vs 月)。

## 5. 波斯向运 Persian Directed（双圈盘 · headline）

**①概述**：黄经象征向运——所有行星/点每年沿黄经 +速率(默认 1°/年)，**本命宫头固定不动**；看向运星触及本命星/宫头。UI=「主限法盘」式双圈(内圈本命/外圈向运)。
**②算法**：`directed_lon(p)=norm360(natalLon(p)+age*RATE)`，RATE={persian:1.0, prophected:30.0, naibod:0.9856473}。应期 hit-list(纯前端算术)：对 mover×target×aspect×方向，`arc=norm360(target−natalLon(p))`、`age=arc/rate`、`date=birth+age*365.2422 天`。
**③后端**：**新模块** `astropy/astrostudy/symbolicdir.py`（镜像 solararc 旋转+相位网；差异：弧=象征弧、houses/angles 不转、pars 跟转 +arc、向运星对本命宫重挂）。`perpredict.getPersianDirectedByDate`。`webpredictsrv.py` 加 `persianchart`。Java `/persianchart` + `getParams` 加 `rateKey` + `AstroHelper.getPersianChart` + 常量。**重编 jar**。双圈盘走 `AstroDoubleChart`（核实：外圈只读 `dirChart.chart.objects+lots`、宫架取本命，故宫头不动即正确）。
**④前端**：`components/astro/AstroPersianDirected.js`（克隆 AstroSolarArc 双圈盘 + 速率下拉 + 前端应期 hit-list `SmallTable`）。Tab `波斯向运/persiandirected`。
**⑤AI**：key=`persiandirected`，段头 `[波斯向运（Persian Directed）]`，挂载随盘同步式(hit-list 算术)。
**⑥验证**：preview 双圈盘(svg 75)随时间重绘且宫头不动、应期 hit-list 有日期；手算金10°金牛+25=5°双子。

## 6. 赤纬推运 Jayne's Progressions（表）

**①概述**：Charles Jayne 赤纬推运——推运后看赤纬平行/反平行。
**②算法**：推运日同 `progression_date`；赤纬 `swisseph.calc_ut(jd,id,FLG_SWIEPH|FLG_EQUATORIAL)[0][1]`；`|d_a−d_b|≤orb`=平行、`|d_a+d_b|≤orb`=反平行(orb≈1)。
**③后端**：`astroextra.py` 新 sibling `build_declination_progressions`（**不动 build_progressions**）+ `swe_decl`/`decl_parallels`。`webastroextrasrv.py` 加 `jaynesprog`。Java `AstroExtraController /jaynesprog` + `AstroHelper.getAstroExtraJaynesProg` + 常量（OPTIONAL_KEYS 已含 targetDate/orb）。**重编 jar**。
**④前端**：`components/astro/AstroJaynesProgressions.js`（克隆 AstroProgressions，列改 赤纬/平行/反平行）。Tab `赤纬推运/jaynesprog`。
**⑤AI**：key=`jaynesprog`，段头 `[赤纬推运（Jayne Declination）]`，async builder fetch `/astroextra/jaynesprog`。
**⑥验证**：preview 赤纬 + 平行/反平行表 + 二/三/小限渲染。

## 7. Balbillus（AstroChart + 削减期表 · 独立引擎）

**①概述**：经典 Balbillus / 129-year Time-Lord 变体（**已订正：非十年大运月制族**）。**独立引擎 `utils/balbillus.js`，不碰 decennials.js（铁律）**。
**②算法**：七星 Balbillus 小年(日19/月25/土30/木12/火15/金8/水20=**129**)；主限长度 `= N×(1 − d/360)`，d=本命星黄经离**擢升度**角距（`nearest` 最近角距 / `forward` 顺黄道距，**做成参数**）；主限序=七星按本命黄经升序、从**起始星**旋转；每主限再以该期主星为起点按 **129 权重**递归切子限(可 5 层)；日期用 Hellenistic **360 日年**(可切 solar 365.2422)。
**③后端**：无。`balbillus.js`：`BALBILLUS_YEARS`/`BALBILLUS_EXALT`/`balbillusDistance`/`balbillusPeriodYears`/`zodiacalOrder`/`buildBalbillusContext`/`buildBalbillusRoots`/`buildBalbillusChildren`/`buildBalbillusSnapshotText`。
**④前端**：`AstroBalbillus.js`——antd `Tree` 懒加载(`loadData` 逐层算子限) + 起始星/年制/距离口径三选项 + 自管快照。Tab `Balbillus/balbillus`（**已恢复显示**）。
**⑤AI**：key=`balbillus`，段头 `[Balbillus]`，挂载随盘同步式，入 PLANET_INFO Set；快照=展平一级主限+二级子限表 `buildBalbillusSnapshotText(chartObj,opts)`。
**⑥验证**：preview 实测——主限序=黄经序旋转、`N×(1−d/360)` 削减(逐星手算对齐)、129 子限和=父期、360日年日期、懒展开正确；mode 默认 nearest，精校待 core 同**本命**盘(其面板经度是 transit 盘、勿误用)。

**④修法**：把 `genAntisciasDom` 逻辑移植进 `AstroAspect.js`，三段后增第四段，**映点(`chart.antiscias.antiscia`)+反映点(`.cantiscia`)两数组都渲染**。过滤改 OR-显示(任一端在显示集即显;两集合皆空→全显,对齐 AstroInfo)——因映点常落主星↔冷门体、AND 会全空。数据已在 payload `perchart.py:getAntiscia()`，**零后端**。
**⑥验证**：preview 相位页底部映点(4)+反映点(2)两段、明暗双模式适配、语义色未变；与 3D 盘信息 tab 逐条一致。
