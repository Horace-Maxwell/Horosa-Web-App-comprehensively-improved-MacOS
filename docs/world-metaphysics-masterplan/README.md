## 目标定位

星阙不应只是“会排盘的工具”，而应成为四层合一的玄学知识与研究平台：

1. 世界玄学计算平台：为东西方命理、择时、占卜、风水、历法、卡牌、灵数、手相面相等体系提供可复核的排盘与规则引擎。
2. 世界玄学知识平台：把典籍、流派、术语、规则、案例、争议、适用边界结构化成可检索、可引用、可教学的知识库。
3. 世界玄学教学平台：提供从入门、进阶、流派比较、案例推演到考试认证的学习路径。
4. 世界玄学证伪/证实平台：将玄学断语转化为可记录、可盲测、可统计检验的预测协议，允许传统经验与现代研究共存。

这个定位需要同时尊重两件事：传统术数内部的复杂规则，以及现代软件工程对可解释、可复现、可测试的要求。顶级产品的核心不是“把所有术法混在一起”，而是给每一种术法保留它自己的世界观、输入边界、计算链路、解释范式和验证口径。

## 顶级标准

星阙的最高标准应包含以下工程与学术要求：

1. 所有计算结果可追溯：输入、时区、历法、真太阳时、地理坐标、天文历表、规则版本、流派配置都应进入结果元数据。
2. 所有流派可并列：同一命盘允许按不同流派重算，并显示差异原因，而不是强行给出单一“正确答案”。
3. 所有断语可引用：AI 或规则引擎输出的判断应尽量关联到规则条目、古籍原文、现代教材或统计模型。
4. 所有预测可记录：预测对象、时间窗、命中标准、置信度、解释变量、事后结果都可写入研究数据库。
5. 所有知识可教学：每个术法应有“概念层、规则层、案例层、练习层、考试层”。
6. 所有体验可专业化：专家需要密度、快捷键、批量盘、对照盘；新手需要引导、术语解释、可视化教学。

## 未来三阶段

### 第一阶段：把已有强项做到“专业可复核”

重点补齐现有模块的规则透明度、流派配置、输出结构化：

- 占星：完整配置中心、古典/现代/印度/中式星法拆分、推运搜索器。
- 八字：格局/旺衰/调候/盲派/象法可切换，命例库与断语来源。
- 印占：Dasha、Nakshatra、Ashtakavarga、Jaimini、Tajika、Muhurta。
- 三式：奇门、六壬、太乙各自成为独立强模块，同时保留三式合参。
- AI分析：从聊天工具升级为“引用型玄学研究助理”。

### 第二阶段：把世界体系纳入统一知识与插件框架

新增塔罗、灵数、符文、Geomancy、Mayan、Tibetan、Vastu、I Ching、姓名学、相术、梦占等模块，但每个模块都遵循统一接口：

- `input_schema`
- `calculation_trace`
- `tradition_profile`
- `interpretation_rules`
- `evidence_protocol`
- `teaching_units`

### 第三阶段：建立可验证玄学研究基础设施

建立匿名化命例库、事件本体、预测登记、双盲评测、统计检验、模型对照和研究报告系统。此阶段的关键不是预设“玄学为真”或“玄学为假”，而是让任何具体规则有机会在透明数据和严格协议下接受检验。
